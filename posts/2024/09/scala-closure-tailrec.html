Scala入门 闭包、尾递归、控制抽象

发表于 2024-09-25 分类于Scala

Scala入门 闭包、尾递归、控制抽象 ​

闭包 ​

闭包这个概念在大一上python的时候就有提及过，但是当时没有认真学，理解的也不是特别好。今天视频老师讲的也不是很清楚，但是我在博客上看到一个比较好的举例，通过这个例子我们可以看到闭包的一个优点：内部函数可以直接访问外部函数的变量，防止变量在函数作用域外被更改。

缺点则是：内存中要一直保存着从外部到内部所有的变量，滥用会对内存造成影响。

尾递归 ​

尾递归对比传统递归的优点：尾递归则直接覆盖栈，每层调用时不用等下一层结果出来才能返回值，每次调用时只有一个栈帧，效率比较高。

案例 ​

一个球从高处落下每次弹起高度为上一次的1/2，求弹起总高度。

自定义一个while循环 ​

这是一个简单的应用案例，但是涉及的内容比较多（闭包、递归、控制抽象），前两个Java中也无法实现，第三个在该案例是一个容易犯错的地方。

实现步骤 ​

该案例要求实现while关键字同样的效果，首先明确while有两个部分：条件和具体代码块。从这个这里我们就确定了要定义两个函数，要使用闭包。

然后我们就需要确定这两个函数的入参：

  • 第一个函数入参是一个布尔值作为循环的条件，但是在while中条件的布尔值是不断变动不是死的，由此可以确定要的是返回布尔值的代码块
  • 第二个函数入参则是一段没有返回值的代码块

具体逻辑交给用户。

核心步骤 ​

  1. 闭包 - 确认入参和返回值（重点）
  2. 内部函数进行整个函数的尾递归
  3. 返回内部函数对象给外部调用

代码实现 ​

scala
// 定义控制抽象实现自定义while循环
object CustomWhile {
  
  // 使用传名参数实现条件判断
  def whileLoop(condition: => Boolean)(body: => Unit): Unit = {
    if (condition) {
      body
      whileLoop(condition)(body)  // 尾递归调用
    }
  }
  
  // 使用闭包实现计数器
  def makeCounter(): () => Int = {
    var count = 0
    () => {
      count += 1
      count
    }
  }
  
  // 尾递归计算弹球总高度
  def bounceHeight(initialHeight: Double, times: Int): Double = {
    @annotation.tailrec
    def loop(n: Int, currentHeight: Double, total: Double): Double = {
      if (n <= 0) total
      else loop(n - 1, currentHeight / 2, total + currentHeight)
    }
    loop(times, initialHeight, 0)
  }
}

总结 ​

  • 闭包：允许内部函数访问外部函数的变量，实现数据封装
  • 尾递归：优化递归性能，避免栈溢出
  • 控制抽象：通过传名参数实现灵活的流程控制
