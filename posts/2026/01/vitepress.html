vitepress使用总结

发表于 2026-01-12 分类于前端

Vite press站点搭建 ​

配置路径别名 ​

typescript
export default defineConfig({
    srcDir: "posts",
    vite: {
        resolve: {
           alias: {
                '@': path.resolve(__dirname, './theme'),
            },
        },
    },
})

整合tailwind4.x ​

1、插件配置

配置方法和其他项目差不多选项都写这个vite里就行

typescript
import tailwindcss from "@tailwindcss/vite";
export default defineConfig({
    srcDir: "posts",
    vite: {
        plugins: [tailwindcss()],
        resolve: {
           alias: {
                '@': path.resolve(__dirname, './theme'),
            },
        },
    },
})

2、引用样式

css
@import "tailwindcss";
@plugin "@tailwindcss/typography";

容易出现问题：配置后样式不生效，可能原因是路径有问题tailwind忽略了.vitepress目录里的文件

解决方法：用source指令

css
@source '../../../posts/**/*.md';
@source '../**/*.{vue,ts}';

文章数据加载 ​

用vitepress提供的api可以递归匹配文件夹下所有文章

更多用法可以参考：构建时数据加载 | VitePress

typescript
import { createContentLoader } from 'vitepress'


export default createContentLoader('../pages/posts/*/*/*.md', {
  transform(rawData) {
    return rawData.map((item) => ({
      id: item.frontmatter.id || 0,
      title: item.frontmatter.title || item.url,
      summary: item.frontmatter.summary || '',
      cover: item.frontmatter.cover || '',
      createTime: formatDate(item.frontmatter.createTime),
      categoryName: item.frontmatter.categoryName || '',
      tags: item.frontmatter.tags || [],
      url: item.url
    })).sort((a, b) => {
      return new Date(b.createTime).getTime() - new Date(a.createTime).getTime()
    })
  }
})

容易出现问题：

在页面中导入时会出现一个Instead change the require of index.js in posts.data.ts to a dynamic import() which is available in all CommonJS modules的错误，这个是运行环境的问题

解决方法

package.json设置,或者把这个ts文件改成mts、cts都是可以的：

json
{
  "type": "module",
}
