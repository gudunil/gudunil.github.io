STM32独立看门狗IWDG详解

STM32独立看门狗 IWDG ​

一、先画图（不查资料，画不出来 = 没掌握） ​

数据从哪来，到哪去？ ​

LSI (~40kHz) ──► 预分频 ──► 递减计数器 (12bit, 从重装值往下减)
                   ↑              │
                IWDG_PR           │ 减到 0？
                                  ▼
                         是 → 产生复位（系统重启）
                         否 → 继续倒计时

喂狗：往 IWDG_KR 写 0xAAAA → 计数器重新装值 → 继续倒计时

引脚怎么接？ ​

无外部引脚！IWDG 是纯内部模块。
时钟源：LSI 内部 RC 振荡器（~40kHz，不精确）

一句话说清楚 + 类比 ​

看门狗的本质是一个自减计数器，溢出后产生复位信号。IWDG 是"倒计时炸弹"：上电后开始倒计时，你必须在倒计时结束前去"喂狗"（重置计时器），否则它就把整个系统复位重启。就像给懒人设闹钟——你每隔一段时间必须按一下"我还活着"，否则闹钟就把你叫醒（复位）。

二、笔记正文 ​

模块概述 ​

项目内容
时钟源LSI（~40kHz，内部 RC，不精确）
计数器12 位递减（0~4095）
超时时间预分频 × 重装值 / LSI 频率（最大约 26 秒）
核心功能程序跑飞/卡死时自动复位
关键特性独立于主时钟，主时钟挂了也能工作

关键寄存器 ​

寄存器作用常用操作
IWDG_KR键寄存器0x5555→解锁PR/RLR，0xAAAA→喂狗，0xCCCC→启动
IWDG_PR预分频寄存器分频系数 4/8/16/32/64/128/256
IWDG_RLR重装载寄存器12 位，计数器从这个值开始往下减
IWDG_SR状态寄存器PVU（PR 可写）、RVU（RLR 可写）

超时时间计算 ​

T_IWDG = T_LSI × PR预分频系数 × (RL + 1)
                                       ^^^^
                               计数器从 RL 减到 0，共 RL+1 个刻度

示例：PR=2(分频16), RL=2499
T_IWDG = 25μs × 16 × 2500 = 1000ms

PR 取值对照：
  0 → /4      4 → /64
  1 → /8      5 → /128
  2 → /16     6 → /256
  3 → /32

初始化流程 ​

c
// 步骤 1: 取消写保护
// 做了什么：IWDG_WriteAccessCmd(ENABLE) → 往 KR 写 0x5555
// 为什么：PR 和 RLR 默认写保护，要先解锁

// 步骤 2: 设置预分频
// 做了什么：IWDG_SetPrescaler(...)
// 为什么：决定"滴答"的频率

// 步骤 3: 设置重装载值
// 做了什么：IWDG_SetReload(...)
// 为什么：决定倒计时多久

// 步骤 4: 喂一次狗 + 启动
// 做了什么：IWDG_ReloadCounter()（写 0xAAAA），IWDG_Enable()（写 0xCCCC）
// 为什么：先装填弹药，再扣扳机

核心操作 ​

喂狗 ​

c
//重装寄存器喂狗

完整初始化 ​

c
// 代码 + 注释

IWDG_SetPrescaler(IWDG_Prescaler_16);

IWDG_SetReload(5000 - 1);

三、自测区 ​

能回答吗？ ​

  1. IWDG 的时钟源是什么？为什么不精确？

    答：LSI 内部 RC 振荡器，约 40kHz。RC 振荡器受温度和电压影响，频率不精确。

  2. 喂狗的本质操作是什么？（写什么值到哪个寄存器？）

    答：往 IWDG_KR 键寄存器写 0xAAAA，计数器重新装载 RLR 的值。

  3. IWDG 启动后能关闭吗？为什么？

    答：不能。一旦启动，IWDG 只能被复位清除，这是为了安全考虑——防止程序跑飞后误关闭看门狗。

  4. 喂狗间隔超过超时时间会怎样？

    答：计数器减到 0，产生复位信号，系统重启。

能默写吗？ ​

c
// 默写区：IWDG 初始化 + 喂狗
void IWDG_Init(void)
{
    // 1. 解锁写保护
    IWDG_WriteAccessCmd(ENABLE);
    
    // 2. 设置预分频
    IWDG_SetPrescaler(IWDG_Prescaler_16);
    
    // 3. 设置重装载值
    IWDG_SetReload(5000 - 1);
    
    // 4. 先喂一次狗
    IWDG_ReloadCounter();
    
    // 5. 启动看门狗
    IWDG_Enable();
}

// 主循环中定期喂狗
void main(void)
{
    IWDG_Init();
    
    while (1)
    {
        // 正常业务代码
        do_something();
        
        // 喂狗（必须在超时前完成）
        IWDG_ReloadCounter();
    }
}

四、使用注意事项 ​

  1. IWDG 一旦启动无法关闭，设计时要考虑是否真的需要看门狗
  2. 喂狗间隔要小于超时时间，建议留出 50% 余量
  3. 在中断中喂狗要谨慎，如果主程序卡死但中断还在运行，看门狗就失去意义了
  4. 调试时可能需要禁用，否则断点停留太久会导致复位
